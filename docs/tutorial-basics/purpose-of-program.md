# 常见计算机语言

## 一、何为计算机语言

### 1、从历史谈起

现在你所能够用到的计算机，也即是广义上的各种电子设备，手机，平板，电脑，其实已经是一个相当完善，相当复杂的系统了。但是这恰恰不太方便我们举例子，以此来理解计算机语言到底是什么，有什么作用，所以我们需要把目光收回到几十年前，那个计算机还是最初级状态的年代。

在现在，你想要计算101*123，其实是相当简单的事情，找到手机里的计算器，输入 101 * 123 然后就能得到结果。但是实际上，你仔细想一想就不对劲，计算机是由电力和机械驱动的装置，它要如何才能进行这样复杂的计算，你甚至可以求50! （五十的阶乘）这种极为复杂的运算，总不能是计算器那头是个人形高斯，时时刻刻给你传递结果吧，所以这里就引出一个疑问，计算机如何处理？

那么，我们不妨以最原始的方式理解计算机，假设有一台十分老旧的计算机，它十分废物，只有如下能力：

- 能够接受一个数字的输入

- 能够一行一行执行某些指令

- 指令仅仅包含加法，对比值是否相同，跳转，创建一个为0的计数器

- 能够在显示器上输出某个数字

单看上述文本，你可能会觉得这个计算机什么也做不到。但是如果我告诉你，这个计算机实际上不仅可以做到加法，还能做到减法，乘法。

首先，如果我希望4+3，计算机可以这么做：

- [1]  接收新的输入: 4

- [2]  接收新的输入: 3

- [3]  执行加法 : 4 + 3 = 7

- [4]  输出结果: 7

那么，如果我希望4-3呢？

- [1]  接收新的输入: 4

- [2]  接收新的输入: -3（这里直接输入一个负数）

- [3]  执行加法 : 4 + (-3) = 1

- [4]  输出结果: 1

同样的，对于乘法，我用4*3举例:

- [1]  接收新的输入: 4

- [2]  接收新的输入: 3

- [3]  创建临时计数器(当前值：0)

- [4]  执行加法 : 4 + 4 = 8

- [5]  执行加法 : 计数器 + 1

- [6]  执行对比 : 对比计数器和输入3的值是否相同

- [7]  执行跳转 : 结果不相同，跳转到指令[4]继续执行

- [8]  输出结果: 12

相当于说，这里其实用反复的加法实现了乘法，同理你也大概能想到，怎么用反复的减法实现除法，甚至于说用反复的乘法实现幂运算。

那这样就引入了一个很有趣的现象，也就是，复杂逻辑是由简单逻辑构成的。这就是计算机语言存在的意义的关键，你所想要实现的复杂功能，都能够通过每一步简单的逻辑实现，而合理设计这个实现逻辑的过程，也就是编程。而这个逻辑本身，或者大白话讲，就是这个步骤的具体格式，具体内容，就是计算机语言。


## 二、计算机语言的分类
### 1、从差异上谈起

你可能已经注意到了，本手册大多数的程序范例都是用了python语言，少数使用了c或者cpp(c plus plus，也就是c++)。

这其实是为了让毫无经验的小白快速入门，因为在所有编程语言中，python确实是比较好入手的那一种，我们举一个很简单的例子看一看

对于python，在屏幕上输出一个Hello World!的步骤是：

```python
print("Hello World!")
```

对于cpp就不那么简单了

```cpp
#include <iostream>
using namespace std;
 
int main() 
{
    cout << "Hello, World!";
    return 0;
}
```

你肯定很懵逼，python这个print是多么的简明，美妙，笔者在此认为，你但凡会点英文，你都会python了，但是相反cpp这个例子，用了好几行，还有很多看似意义不明的语句。

举例而言，在cpp里，你可以看见include，你可以看见int，你可以看见main，你可以看见cout，你可以看见莫名其妙的return 0，这些东西和你要输出的"Hello World!"看似没什么关系。但这到底有什么作用呢，这里笔者暂时不提，有兴趣的同学可以自行搜素。如果你不能理解，就当成一个固定格式先记住吧。

而有关为什么差异这么大，这个例子中，关键的原因是，cpp是编译语言，python是解释语言，这一点感兴趣的朋友们可以了解一下，总之，每个语言都有它自己的语法，自己的一些"规矩"，这一点上，是有所区别的。有些语言就是更复杂，有着更头痛的语法，有的则更加简约。所以要接收这种语言上的差异，才能让自己的开发能力更上一层楼。


### 2、在需求上的不同

每一门语言，基本上都对应着一种特殊的需求。

当初c语言被发明出来，就是因为汇编的冗杂性，实在是难以学习，而cpp的出现则是为了解决c语言过于底层，缺少例如class（类）的设计，而csharp的出现则是为方便程序员在windows平台下快速开发出带有GUI的软件，java和kotlin则为Android系统软件的开发提供了便利。

但是如果有人因为发展史上存在这么一种先后顺序，就立马认为：cpp就是比c语言好，csharp就是比cpp好，这是纯属扯犊子。因为即使是现在2025年了，cpp语言依然在深度学习，算法设计等等领域，有着广泛应用，即使是c语言，在嵌入式开发领域，也有不可取代的地位。

因此，在不同的需求上来看，大致可以如此分开。你看不懂也没啥，也别想着全学明白，这只是做一个整体的展示

---

#### 一、系统与底层开发
- C：操作系统、嵌入式系统、驱动开发  
- C++：图形引擎、浏览器内核、数据库内核  
- Rust：安全系统编程、嵌入式、区块链  

#### 二、网页与前端开发
- JavaScript / TypeScript：前端逻辑、框架、Node.js 服务端  
- HTML / CSS：页面结构与样式  
- WebAssembly：高性能前端模块  

#### 三、后端与企业应用
- Java：企业后台、银行系统、分布式服务  
- C#：桌面软件、Unity 脚本、企业信息系统  
- Go：微服务、云原生系统、网络服务  

#### 四、数据科学与人工智能
- Python：数据分析、机器学习、自动化  
- R：统计分析、可视化、生物信息学  
- Julia：科学计算、仿真建模  

#### 五、移动开发
- Swift：iOS、macOS 应用  
- Kotlin：Android 原生、服务器开发  
- Dart：Flutter 跨平台应用  

#### 六、游戏开发
- C++：3D 引擎、性能模块  
- C#：游戏脚本（Unity）  
- Lua：游戏脚本逻辑  
- HLSL / GLSL / Metal：图形着色器  

#### 七、脚本与自动化
- Bash / Shell：Linux 自动化、部署脚本  
- PowerShell：Windows 系统管理  
- Perl：文本处理、系统脚本  
- Python：自动化、DevOps、爬虫  

#### 八、数据库与查询语言
- SQL：数据查询与建模  
- PL/SQL / T-SQL：存储过程与触发器  
- GraphQL：Web API 查询  
- MQL：NoSQL 查询（MongoDB）  

#### 九、安全与逆向
- Assembly：二进制逆向、漏洞分析  
- C / C++ / Rust：安全工具开发  
- Python：渗透测试脚本  

#### 十、科学与工程计算
- Fortran：数值仿真、气象建模  
- MATLAB：工程计算、信号处理  
- Julia：高性能科学计算  

#### 十一、硬件与嵌入式
- Verilog / VHDL：芯片逻辑设计  
- C / C++：单片机编程  
- Rust / MicroPython：嵌入式控制  

#### 十二、网络与分布式系统
- Go：并发服务、分布式系统  
- Erlang / Elixir：容错系统、电信平台  
- Python：网络爬虫、API 自动化  

如果你想着一口气把以上内容全部学完，要么你是天才，要么你是疯子，我们学计算机的，一定要灵活，技能一定是实时跟随需求变化的，所以需要什么学什么，才是应该有的，聪明的方案。

### 3、有关计算机语言的补充

在学习计算机语言的过程中，你可能会听到有人说：“我会Cpp，c，python，java...”，但这真的重要吗？仔细想想，其实掌握多少门语言，其实并不能说明这个人有多会编程，而掌握了学习语言的能力，掌握了工程思维，以及习得如何快速分析问题的能力，才是真本事啊。

而有关计算机语言的实现，我觉得如果你不是什么特别狂热的爱好者，在这个阶段是不必了解原理的，只需要了解如何使用即可。等某一天你真的很好奇背后的逻辑，你自然会去学习冯诺依曼架构，汇编语言，编译原理之类的东西，这里笔者就不再赘述了。

以及，虽然有论调说：“AI取代程序员，自然语言编程”等等，笔者在此承认，随着诸如GPT类似的LLM不断发展，大量基于文本的重复性coding确实可以被AI代替，但是那是局部的，精细的，也就是说，如果你并不懂得如何编程，如何合理设置架构，如何合理控制流程。即使是有GPT100帮助你，你也不一定能写出什么东西来。正所谓GPT有，DeepSeek有，不如自己有，打好编程基础，学会工程思维，才是成为一个好的程序员的本分。
