# 程序的用途

## 一、程序是什么

### 1、只缘身在此山中

在当今社会中，随着科技的不断发展，程序作为一种附随着科技而存在的事物，已经广泛地存在于各种地方，但是如果你不深入思考，恐怕也很难想到其全面性和复杂性。举例而言，你有想过手机流量的消耗计算吗，你有想过支付系统为什么难以破解吗，你有想过LLM（大语言模型）何以思考吗？其实如果只停留于“使用”的层面，那当然是不难掌握的，可要是到了“理解”的层面，你一定会感叹于程序之美妙了。

根据百度百科的定义，程序可以被如此描述
> 程序（Program）是指为得到特定结果而由计算机等具有信息处理能力的装置执行的代码化指令序列。

这是个极为笼统而抽象的描述，但如果用大白话来说，就是，给计算机下指令，以此达到你的目的。这目的可以是付钱，可以是发消息，可以是看视频，只要指令合理，清晰，计算机就可以帮你做到这一切。

从表面上看，程序似乎只是在告诉计算机做什么，但是这就引入一个问题，程序何以做到？程序员又如何构建程序，这也就是我们需要思考的一个问题。


## 二、数据与流程

对于程序本身来说，它只包含两大要素：**数据（Data）与流程（Process）**

 > 数据：程序运行时需要处理的信息，它以变量、常量、数据结构（如数组、链表、哈希表）等形式存在。精妙的数据设计能让程序既高效又易维护。

 > 流程：数据在时间维度上的变化与流动，通常由控制结构（顺序、分支、循环、并发）来实现。流程决定了事件先后、条件判断以及操作的重复与终止。

定义是枯燥的，说大白话就是，**通过执行流程处理数据**,这一点并不难理解，毕竟对于python来说。即使是最基础的
```python
print("Hello World!")
```
就这个最简单的例子而言，print()这个功能调用了"Hello World!"这个数据，这就刚好是我们说的执行流程处理数据。

不过如果你还是没太能理解这个东西，也不必担忧，因为这个东西是在反复的实践中才能认知到的，所以现在你只需要大致留个印象，在日后的开发中，说不定某一时刻就会大彻大悟了。

## 三、想法与逻辑的桥梁

程序的根本目的，是把人的想法转化为计算机能理解的逻辑，这也是程序员的任务。这个过程包含三个核心步骤：

表达需求：你想让电脑做什么？

拆解步骤：实现这个目标需要哪些明确的步骤？

转化语言：用某种编程语言把这些步骤告诉机器。

这听起来似乎很简单，但是实际上，在一个需求背后，程序员往往要考虑的不仅要有数学逻辑上的处理，还有一些给计算机兜底的措施，因为在人的想法里，有一些东西是默认的，比如死人不能吃饭，但是计算机怎么会知道死人不能吃饭呢？毕竟对它来说，死和吃饭没有必要的关系，在计算机眼里，“死”是一个状态，“吃饭”是一个行为，这本质上并不冲突。所以程序员还得在这里做一个兜底，在执行“吃饭”这个行为前，还要判断这个人是否处在“死”这个状态。

那么笔者在此直接使用代码举例，在中华人民共和国，未成年人是指不满十八周岁的自然人，那么假设有一个场景，你需要编写一个程序，使得对于给定的一个岁数age，判断其是否是未成年人，如果是，输出“未成年”，否则输出“成年”。

```python
age = int(input("请输入年龄："))

if age < 18:
    print("未成年")
else:
    print("成年")
```

看着很美好！这个程序获取用户的输入到了age，然后判断其是否小于18，并输出结果。想必大多数读者都觉得这个代码已经没有修改的必要了。但是假设有一个猴子，他发现了这个程序，然后他输入了
```
-1
```

这个时候，python仍然认为，-1 < 18啊，所以这个-1岁数的人，当然也是未成年人了，可现实生活告诉我们，哪有负数岁数的，所以这个程序并不完善，合理的程序应该是

```python
age = int(input("请输入年龄："))

if age < 0:
    print("不合法的年龄")
else:
    if age < 18:
        print("未成年")
    else:
        print("成年")
```

这就是所谓，程序员在翻译人的想法到程序的想法的步骤，首先是数学和逻辑上的处理，然后还要考虑潜在的隐藏条件，替计算机兜底。而这个程序本身，也恰好模拟了人类根据岁数思考对方是否是成年人的过程。

## 四、复杂而迷人

过于单纯的人很难得到他人的青睐，保持神秘与复杂才会增加人格魅力。同样的道理，放在程序上也是一样的，程序的迷人之处，只有当它复杂的时候，才会体现出来，像是刚刚笔者举得一个例子，实在是程序能做到的事情中，微不足道的一点点。

而这就又要说回本文开头的问题了，那些大型项目，比如支付宝吧，支付宝每时每刻都在处理数以万计的支付订单（也就是它所要处理的数据），那么它的处理逻辑（也就是流程），到底要怎么样设计呢？我们不妨先想象用人来达到这点，在没有移动支付的年代，甚至没有银行自助机的年代，仅仅依靠人来进行转账，那是一件多么痛苦的事情，柜台前排不完的队，还要出示证件证明你是本人，而现在我们有了支付宝，一个二维码就能付钱了，这是一种怎样的效率提升，背后的程序是怎样的复杂，这个时候程序的迷人之处就体现出来了。

而计算机这门学科同样有着这种特质，最外行的觉得她不迷人，因为根本就没有接触过，无知者无畏嘛。稍微懂行一点的就会觉得计算机真是奇妙，像是一片自由广阔的天地，到处都是可以学习，可以探索，可以实践的地方。

而等你真的和计算机熟络了，真的理解了程序不过是一条一条简单逻辑所构成的，那么说不定还会回过头来，感叹的说一句：“也就不过如此嘛。”
