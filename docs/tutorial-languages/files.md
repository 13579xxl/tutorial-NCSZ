## 文件的读写
 文件的读写在程序中必不可少。假设现在有一文件my_file.txt，在python中，我们可以采用如下函数来对文件进行操作：
 ```python
 '''
 open函数用于打开文件，在这里会创建一个实例f，可以看成Python的File类的实例化（在上一章有讲过），这里的with是一个语法糖（即简化程序），可以自动close文件
 '''
with open('./my_file.txt', mode='r+') as f:
    message = f.read()
    f.write("Hello,World!")
print(message)
 ```
 以上就是读写文件的最基本的方法了，read即“读”，write即“写”，r+即“可读可写”，按上述格式即可完成最基本的读写了，open函数的具体格式在此不加阐述（由于内容牵涉过多），感兴趣同学可以自行学习。
 ## 模块——多文件编程
 正如前几节所讲的那样，很多时候我们需要把一些东西进行封装（encapsulation），这样方便我们进行更好的处理。实际上，很多大的项目如果放在一个文件里面写，会变得冗长且不好维护，故而我们可以将每个板块单独写为一个文件，要使用时再调用，就像搭积木那样。这样，整个项目就会变得美观且井然有序
 在python中，实现多文件编程，采用的是“模块(module)”这一概念
 以下描述来自菜鸟教程：
 >Python 模块(Module)，是一个 Python 文件，以 .py 结尾，包含了 Python 对象定义和Python语句。

 简单来说，我们写的.py程序，就是一个模块
 使用这些模块非常简单：
 ```
 import module1[, module2[,... moduleN]]
 ```
 这样，我们就引入了一个模块
 在使用模块内的内容时（比如调用一个函数），我们可以采用如下语法：
 ```
 modname.FunctionName()
 ```
 模块名加函数名的方式即可
 一般的，如果我们不想要引入所有内容，可以使用from-import语句
 ```
 #从指定的modename导入
from modname import name1[, name2[, ... nameN]]
 ```
 这样就可以导入指定的模块了，当然，如果导入所有模块也可以使用这一语句，实际上这也是很常见的用法：
 ```
 from modname import *
 ```
