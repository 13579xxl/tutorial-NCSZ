## 从零开始

每个程序员都有一段充满着迷茫和兴奋的探索期，就像大航海时代的水手们。计算机作为一个工具，作为一个平台，拥有着无限的可能性。但是这种可能性也能带来许多困惑和不解，所以本文，笔者以一个不那么严谨，略带玩笑的叙述风格，希望能够为你提供一些可借鉴的经验。

## 计算机

说到计算机，在中文上，见字如意地看，就是很纯粹的，计算的机器。但是计算到底是什么呢？那就要回到英文中。

计算机的英文是computer，从词根上拆开来说，就是com put er，而com有整体，整个的意思，这里其实就是in和out的总称。而input，output，恰好就是计算最关键的东西，输入的数据和输出的结果，毕竟计算本身只是一个行为，没有用于计算的数据和计算后的结果，计算本身就失去了意义。

所以综上我们就知道了，其实计算机所做的事情，就是获取一个输入，反馈一个结果，这个输入可以是你点击一下鼠标，反馈就是游戏里的枪执行一次开火，也可以是输入一个数字，反馈就是它的平方根。而程序员主要的任务就是，想方设法地设计一个计算流程，使得对一个输入，返回一个结果。

## 编程

讲完计算机是什么，那么编程又是什么呢？

假设你是一个交通指挥员，现在在你面前有一个傻子驾驶员，这个傻子驾驶员只能非常笨拙地执行前进，后退，左拐，右拐。而你需要指挥它从南昌十中开车到旭辉广场的停车场。那么一个好的程序员，应该已经马上分析出了这个问题的输入，过程，和结果应该是怎样的:

- 输入：一个车，一段路径

- 过程：一套指挥指令，包括前进长度，拐弯方向

- 结果：车要停在路径的结尾

所以，现在就是来设计过程了，那么我来进行一波操作:

- 向前3米

- 向右拐弯

- 向前50米

- 向左拐弯

- 向前10米

虽然我不能保证这个车完美的停在了停车位上，或者完全遵守了交通规则。但是至少它大概在停车场里了，也就是完成了编程的要求，那就行了，毕竟在这个场景下，又没有这两个条件的结果上的要求。

日常生活中请遵守交通规则！

所以这就是一个程序员基本上在做的事情了，找出输入，结果，设计中间流程。其中设计中间流程的这个步骤，我们就叫它编程。

## 实战

听了我扯这么多废话，不知道你有没有听腻呢，如果没有听腻，接下来的东西可就没那么“废话了”

在实际编程中，通常你扮演的仍然是一个最高指挥的角色，你拥有以下能力。

- 申请

- 处理

- 调用

听着很抽象，本质上还是为了输入，输出服务。比如现在你被要求编程一个程序:

```
对于"在所有失去的人中，我最怀念的是我自己"这句话，获取其的长度并打印出来
```

那么首先我们马上可以想到，输入是str，过程是数数，结果是打印的长度

所以我使用python来看，就可以是

```python
str = "在所有失去的人中，我最怀念的是我自己"
length = len(str)
print(length)
```

很清晰的逻辑，第一行，我们定义str暂时存储这句话的内容，毕竟如果你不把这玩意写下来，就像是写数学题不把已知条件写下来。

第二行是调用len(str)，你可能很好奇len(str)是什么意思，数学好的同学，就把它看作像是f(x) = x的长度，这种映射函数吧，如果数学不那么好的同学，你就记住，这是你可以使用的魔咒，写出它就可以获得长度了。

第二行还做了一件事情，申请了一个变量length，暂时存储了结果，为了辅助计算的过程，你作为程序员，可以申请一些“辅助道具”，比如这个变量就像一个临时的背包，装载你刚刚获取的物资。

第三行就是print(length)，这属于最后的打印结果。

如果你已经安装了python，可以直接复制以上内容，你会看见18的结果被输出。没安装也不要紧，你数一下那个字符数量，也确实是18。

## 好像很无聊

好吧，我承认，到这你会觉得挺没意思的，我获取这个字符串的长度意义何在呢？

其实这只是一个非常简短的例子，如果复杂化，可以是：

- 给你一个图片，返回它的分辨率

- 给你一个无序数列，返回它的有序数列 

- 给你三个坐标，返回三条边长

到这你应该慢慢知道了计算机到底是做什么的，其实就真的只是计算而已，像是游戏里的跑动，开枪，其实本质上就是输入是键盘或者鼠标的信号，输出屏幕上变化的像素，这也是计算。

到了这，相比你也大概知道了计算机科学到底是一门怎样的学科，编程又是一种怎样的技能，不知道你看完上文有什么样的感受，是恍然大悟，还是揭开神秘感后的失望，不过这都不重要，只要你保持热情和好奇，你就有掌握这门学科的潜力，但亦不可浅尝辄止，这也只是开始的开始，教程的教程。

那么希望对你有帮助的同时，也祝贺你开启了计算机学科的大门。
